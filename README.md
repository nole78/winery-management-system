# Winery Management System

## Overview

This project is a team-developed application built as part of a university course, simulating a real-world software development process using Agile (Scrum) methodology.

The application is designed to manage winery operations, supporting multiple user roles with different permissions and system functionalities.

---

## Features

* User authentication and role-based authorization
* Multiple user roles with different access levels
* Data storage using XML-based (NoSQL) database
* Separation of concerns across application layers
* Modular and maintainable architecture

---

## Tech Stack

* **Backend:** C# (.NET)
* **Database:** XML (NoSQL approach)
* **Tools:** Git, GitHub, Jira

---

## Architecture

The application follows a layered architecture:

* **Presentation Layer** – handles user interaction
* **Business Logic Layer** – contains core application logic
* **Data Access Layer** – manages XML data persistence
* **Domain Models** – represents core entities

Interfaces and service implementations are used to ensure flexibility and maintainability, following SOLID principles and Clean Code practices.

---

## Development Process

* Developed in a team environment using **Agile (Scrum)**
* Tasks and progress tracked using **Jira**
* Version control managed with **GitHub**

---

## Learning Outcomes

* Practical experience with team-based software development
* Understanding of Agile workflows in real projects
* Applying Clean Code and SOLID principles
* Designing layered applications with clear separation of concerns
